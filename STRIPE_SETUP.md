# Stripe Payment Integration Setup

This document describes the complete Stripe payment integration for the Nanowork wallet system.

## Overview

The payment system consists of:
1. **Frontend (Wallet.tsx)**: Stripe Elements UI for card input
2. **Worker API (apps/worker/src/routes/wallet.ts)**: Creates PaymentIntents
3. **Webhook Handler (apps/worker/src/routes/stripe-webhooks.ts)**: Processes successful payments and credits accounts

## Architecture

```
User clicks "Buy now"
  → Frontend calls /api/wallet/topup
  → Worker API creates PaymentIntent with Stripe
  → Frontend shows card input modal (Stripe Elements)
  → User enters card details and submits
  → Stripe processes payment
  → Stripe sends webhook to /api/stripe/webhooks
  → Worker webhook handler credits user account
  → Frontend refreshes balance
```

Note: The API runs on Cloudflare Workers using Hono framework. In development, it runs on port 8787.

```bash
STRIPE_WEBHOOK_SECRET=whsec_...          # Generated by Stripe CLI or Dashboard
SUPABASE_URL=https://...supabase.co
SUPABASE_SERVICE_ROLE_KEY=...
```

## Stripe Dashboard Setup

### 1. Get API Keys
1. Go to [Stripe Dashboard](https://dashboard.stripe.com)
2. Navigate to **Developers → API keys**
3. Copy the **Publishable key** (for frontend)
4. Copy the **Secret key** (for backend)

### 2. Set Up Webhook

#### Option A: Development (Stripe CLI)
```bash
# Install Stripe CLI
brew install stripe/stripe-cli/stripe

# Login to Stripe
stripe login

# Forward webhooks to local worker
stripe listen --forward-to http://localhost:8787/api/stripe/webhooks

# Copy the webhook signing secret (whsec_...)
# Add it to apps/worker/.dev.vars as STRIPE_WEBHOOK_SECRET
```

#### Option B: Production (Stripe Dashboard)
1. Go to **Developers → Webhooks**
2. Click **Add endpoint**
3. Enter your webhook URL: `https://api.nanowork.ai/api/stripe/webhooks`
4. Select events to listen for:
   - `payment_intent.succeeded`
   - `customer.subscription.deleted`
   - `customer.subscription.updated`
   - `invoice.payment_succeeded`
   - `invoice.payment_failed`
   - `checkout.session.completed`
5. Click **Add endpoint**
6. Copy the **Signing secret** (whsec_...)
7. Set it as a Wrangler secret: `wrangler secret put STRIPE_WEBHOOK_SECRET`

## Implementation Details

### Frontend (apps/web/src/dashboard/Wallet.tsx)
- Uses `@stripe/stripe-js` and `@stripe/react-stripe-js`
- Loads Stripe with publishable key
- Shows CardElement in modal for secure card input
- Calls `confirmCardPayment` with client secret
- Refreshes balance after success

### Worker API (apps/worker/src/routes/wallet.ts)
- Validates bundle selection
- Creates Stripe PaymentIntent with amount in cents
- Includes metadata: `userId`, `creditAmount`, `bundle`
- Returns `clientSecret`, `amount`, `credits` to frontend

### Webhook Handler (apps/worker/src/routes/stripe-webhooks.ts)
- Verifies webhook signature with `STRIPE_WEBHOOK_SECRET`
- Handles `payment_intent.succeeded` event
- Extracts metadata from PaymentIntent
- Atomically updates user's credit balance
- Records transaction in `credit_transactions` table

## Database Schema

### profiles table
```sql
- id (uuid)
- credits (integer) -- User's current balance
```

### credit_transactions table
```sql
- id (uuid)
- user_id (uuid) -- References profiles.id
- amount (integer) -- Positive for credits, negative for debits
- balance_after (integer) -- Balance after transaction
- type (text) -- 'topup', 'spend', 'refund'
- description (text) -- Human-readable description
- stripe_payment_intent_id (text) -- Stripe PI ID for top-ups
- created_at (timestamp)
```

## Security Notes

1. **Never expose secret key in frontend code**
2. **Always verify webhook signatures** (prevents fake webhooks)
3. **Use HTTPS in production** (required by Stripe)
4. **Store sensitive keys in environment variables**
5. **Use idempotency** (Stripe handles duplicate webhooks)
6. **Log all payment events** (for debugging and reconciliation)

## Troubleshooting

### Payment fails silently
- Check browser console for errors
- Verify Stripe publishable key is correct
- Check network tab for API errors

### Webhook not received
- Verify webhook URL is accessible
- Check webhook signing secret matches
- Use Stripe CLI `stripe listen` for local testing
- Check webhook logs in Stripe Dashboard

### Credits not added
- Check webhook handler logs
- Verify metadata in PaymentIntent
- Check database transaction logs
- Ensure `addCredits()` function succeeds

### "Stripe not loaded" error
- Verify `VITE_STRIPE_PUBLISHABLE_KEY` is set
- Check for ad blockers blocking Stripe JS
- Verify Stripe publishable key format (pk_test_ or pk_live_)
